Estándares de Desarrollo

Información General

Este documento contiene directrices para aplicaciones creadas por el equipo de trabajo de Tecnología. Se creó y se mantiene en continua actualización con el fin de mantener una base homogenea de desarrollo del departamento.

La motivación principal de este documento tiene dos razones: 1) consistencia de código y 2) prácticas recomendadas. Al mantener la consistencia en los estilos y convenciones del código, podemos aliviar la carga del mantenimiento del código desactualizado y mitigar los riesgos de daño en el futuro. Adhiriéndonos a prácticas recomendadas y reconocidas aseguramos tiempos de carga de página, rendimiento y código mantenible optimizados.

Prácticas Generales

Sangrado

Para todos los lenguajes de código, exigimos que el sangrado se realice mediante tabulaciones suaves (usando el caracter de espacio). Pulsar Tab en su editor de texto debe ser equivalente a cuatro espacios.

Legibilidad vs Compresión

Preferimos la legibilidad a la reducción de tamaño de los archivos cuando se trata de mantener archivos existentes. Se debe usar espaciado en blanco y comentarios para darle una mayor claridad al codigo. No es necesario para ningún desarrollador comprimir el HTML, CSS o JavaScript ya que estos pueden ser comprimidos mediate procesos de optimación.

Comentar código

Si queremos ser condescendientes con el team de Tecnología comentemos nuestro código. Se recomienda comentar los archivos al inicio incluyendo una pequeña descripción o título, autor, fecha de creación y el correo electrónico. Con estos datos cualquier persona podra localizarlos de ser necesario y tendra una leve noción a cerca del código que va a trabajar.

Tambien se debera comentar bloques de codigo que no sean facil de decifrar en poco tiempo y sera del gusto del programador comentar variables.

Idioma

Debera usarse el idioma ingles como estandar para todos los comentarios, variables, funciones, metodos, IDs, classes, selectores, etc..

Convención de código

HTML / CSS / JS

A continación expondremos las conveciones de código que usaremos durante el desarrollo de nuevos proyectos con el fin de tener una consistencia en el equipo de Tecnología.

También cabe mencionar que si adoptamos estas convenciones sera muy familiar trabajar con código hecho por otros compañeros.

HyperText Markup Language (HTML)

  1. todos los tags y atributos deberan ir en miniscula con sus comillas dobles. En caso de que un tag, atributo, id o clase sea formado por dos palabras o mas, estara separado entre cada palabra por un guion medio.
  2. en lo posible usar tags descriptivos (HTML 5).
  3. idioma ingles.

Cascading Style Sheets (CSS)

  1. todos los selectores deberan ir en minuscula. Al igual que el HTML en caso de que un selector sea formado por dos o mas palabras debera estar separado por un guion medio.
  2. nombre de selectores semanticos. No usar adjetivos (color, tamaño, etc.. )
  3. en caso de usar un pre compilador de css (less , sass , stylus) se debera separar en modulos (componentes) el css como lo hace por ejemplo twitter bootstrap.
  4. idioma ingles.

CSS Utilidades

  1. Pre compiladores de css: less , sass ( compass ).
  2. Css frameworks: Twitter Bootstrap, Skeleton, Foundation.

Javascript (js)

  1. Declaracion de varibles al inicio.
  2. Nombrar variables en Camelcase.
  3. Hacer uso de Namespaces si tu javascript se necesita usar en otro lado.
  4. Capitalizacion de constructores.
  5. hacer uso de strict mode.
  6. idioma ingles.

JS Utilidades

  1. DOM manipulacion: jQuery, Backbone, Underscore, Ember, Angular
  2. JS templates: Underscore, Handlebars, Mustache.
  3. Nodejs: Bower, Grunt, Requirejs, Yeoman.
  4. Livereload.
